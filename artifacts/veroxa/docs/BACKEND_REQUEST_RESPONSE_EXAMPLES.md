# Backend Request Response Examples

Use these portal-first examples for 24-hour review/answer/next-step responses. Adjust specifics, but keep boundaries intact.

## Included request

“Thanks — this fits your Complete Online Presence service. Veroxa will review it and prepare the next step. Nothing goes live without Veroxa team review.”

## Needs confirmation

“Veroxa can prepare this after the restaurant confirms the exact details. Please confirm the hours/menu/prices/offer wording before we use it publicly.”

## Add-on available

“This is available as an add-on. A new basic website is +$95, and missing Facebook/Instagram profile creation is +$45/profile. Veroxa can prepare the next step after you confirm.”

## Coming soon

“Yelp, TikTok, Reels/video content, ads management, daily posting, automated publishing, and live integrations are coming soon / not included at launch. Veroxa can note this for later.”

## Not included

“This is outside Veroxa’s launch scope, so Veroxa cannot handle it as part of the monthly service.”

## Customer-service request

“Veroxa does not handle customer-service replies, comments, DMs, inboxes, refunds, complaints, or order questions. The restaurant should respond directly.”

## Full custom website request

“Full custom website development is not included. Veroxa can align/refine an existing site when access is provided, or prepare a simple new basic website as a +$95 add-on.”

## Existing offer/promotion confirmation

“Veroxa does not invent discounts or promotions. If this offer already exists, please confirm the exact details, dates, price, and restrictions before Veroxa prepares public copy.”

## Low media supply

“Posting may slow when usable media is unavailable. Please send clear food, menu, storefront, dining-area, and team-approved photos so Veroxa can keep picture-based content moving.”

## Unclear/risky request

“Veroxa will hold this until Faraz reviews it. Please share any missing context so the team can decide the safest next step.”
