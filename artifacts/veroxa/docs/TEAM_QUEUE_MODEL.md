# Team Queue Model

Team queues should stay consistent, calm, and action-focused. Every queue item
should answer the same operational questions when the data is available:

1. Client / restaurant.
2. Type of work.
3. Current status.
4. Priority or urgency.
5. Due timing, if known.
6. Suggested next step.

## Queue roles

- **Work Queue**: day-to-day client work, media review, content prep, queue or
  hold decisions, and client follow-up.
- **Direction Queue**: client requests and optional direction such as use this
  media, save for later, push a special/event, avoid an item, or general notes.
- **Report Queue**: weekly/monthly report source items and report drafts that
  need Veroxa team review before sharing.

## Empty states

Queues should not feel broken when empty. Use short, useful copy that explains
what will appear there once live work exists. Do not show demo restaurants as
active clients when live account data is not connected.

## Language rule

Team pages may show operational detail, but they should use calm language such
as Ready for review, Needs confirmation, Queue for later, Hold for later,
Suggested next step, and Reports due. Avoid command-center clutter.
