VEROXA FEATURE BUILD — EVIDENCE-BASED SELECTION ENGINE V1 DEMO ONLY

Objective:
Build a demo-only “Evidence-Based Selection Engine” inside Veroxa.

This feature should show that Veroxa makes smarter content and operations recommendations based on daily data, previous performance, content history, media quality, posting schedule, client health, and reporting memory.

This is NOT real AI yet.
This is NOT real memory yet.
This is NOT connected to Supabase yet.
This is a simulated decision engine using fixture/demo data only.

Hard invariants:
- AUTH_MODE must remain exactly "placeholder"
- Portal remains disconnected from real Supabase
- No OpenAI / Anthropic / Gemini / external AI APIs
- No real AI provider
- No Supabase storage upload
- No real publishing APIs
- No payment or billing integrations
- No real client/customer/restaurant data
- Demo gate veroxa-preview remains unchanged
- Roles remain exactly Client, Team, Operator, Owner
- Pricing remains unchanged
- No files added to supabase/migrations
- Use fixture/demo data only

Feature concept:
Veroxa should visually explain:

“Based on what worked before, what the client uploaded today, what is scheduled next, what content is running low, and what the restaurant needs this week, Veroxa recommends the smartest next move.”

Evidence-Based Selection Engine

PART B — Build evidence selection engine logic

Create a pure TypeScript/JavaScript deterministic rule engine.

It should NOT call AI.
It should NOT call API.
It should NOT use database.
It should only compute from demo fixture data.

Engine should export functions like:

- getEvidenceProfile(clientId)
- scoreMediaForNextPost(clientId)
- recommendNextPost(clientId)
- recommendPostingTime(clientId)
- recommendClientAction(clientId)
- recommendOperatorAction(clientId)
- getEvidenceReasons(clientId)
- getEvidenceTimeline(clientId)

Each recommendation should include:
- recommendationTitle
- confidenceScore
- decisionType
- selectedItem
- recommendedAction
- evidenceReasons[]
- riskNotes[]
- nextStep
- demoOnly: true

Example output:
Recommendation:
“Use the grilled platter image for Friday lunch.”

Evidence reasons:
- “Similar lunch posts had 18% higher engagement.”
- “Client has only 6 content runway days left.”
- “Instagram is currently the strongest platform for food visuals.”
- “Friday 11:30 AM matches the highest-performing lunch window.”
- “Image quality score is 91/100.”

Confidence score:
87%

Important:
Make the engine explain itself clearly. The demo should feel evidence-based, not random.
