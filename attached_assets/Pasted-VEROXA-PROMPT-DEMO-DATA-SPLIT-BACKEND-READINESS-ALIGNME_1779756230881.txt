PART 3 — MOVE DATA SAFELY

Move demo data from demoData.ts into the new files gradually.

Rules:

- Preserve export names when possible.
- Preserve TypeScript types.
- Preserve existing object shapes unless a tiny naming cleanup is safe.
- Avoid changing page logic.
- Avoid breaking imports.
- Keep behavior visually the same.

If a data object is used in many places and moving it risks breaking the app, you may leave it in demoData.ts for now and document it as a future split candidate.

Safety matters more than perfection.

------------------------------------------------

PART 4 — CREATE BACKWARD-COMPATIBLE EXPORTS

Important:
Existing pages currently import from:

src/data/demoData.ts

Do not break them.

Use one of these safe approaches:

Option A — Preferred:
Keep src/data/demoData.ts as a compatibility barrel file.

It should re-export data from the new split files.

Example:

export * from "./demo/demoClients";
export * from "./demo/demoMediaAssets";
export * from "./demo/demoPosts";

Option B:
Update imports across the app to import from src/data/demo.

Only use Option B if it is safe and not too noisy.

Preferred result:
Existing imports continue working.
