PART 2 — WEEKLY_REPORTS TABLE PLAN

Document table:

weekly_reports

Fields:

- id uuid primary key default gen_random_uuid()
- client_id uuid not null references clients(id) on delete cascade
- week_start date not null
- week_end date not null
- posts_planned integer not null default 0
- posts_published integer not null default 0
- top_post_id uuid references posts(id) on delete set null
- status text not null default 'drafted' check in ('drafted','validated','published')
- summary_json jsonb
- client_safe_summary text
- internal_validation_note text
- drafted_by_user_id uuid references user_profiles(id) on delete set null
- validated_by_user_id uuid references user_profiles(id) on delete set null
- published_at timestamptz
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

Notes:

- Client only sees published reports through client-safe view.
- internal_validation_note is never client-facing.
- summary_json may contain internal metrics; client-safe_summary should power client view.

------------------------------------------------

PART 3 — MONTHLY_REPORTS TABLE PLAN

Document table:

monthly_reports

Fields:

- id uuid primary key default gen_random_uuid()
- client_id uuid not null references clients(id) on delete cascade
- month_key text not null
- status text not null default 'drafting' check in ('drafting','operator_review','approved','published')
- summary_json jsonb
- client_safe_summary text
- internal_operator_note text
- approved_by_user_id uuid references user_profiles(id) on delete set null
- published_at timestamptz
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

Constraint:

unique(client_id, month_key)

Notes:

- Monthly reports require operator approval before publishing.
- Client sees only published monthly reports through client-safe view.
- Owner/operator can see internal notes.
- Team can draft but not final approve unless future policy allows.
