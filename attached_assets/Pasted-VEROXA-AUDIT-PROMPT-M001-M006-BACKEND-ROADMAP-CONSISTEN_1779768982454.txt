PART 4 — RLS CONSISTENCY AUDIT
------------------------------------------------

Audit RLS logic across all drafts.

Confirm role rules stay consistent:

Client:
- sees only own data
- sees client-safe views where needed
- cannot see internal notes
- cannot see raw drafts/concepts
- cannot see activity_logs raw table
- cannot see owner revenue
- cannot update pricing or internal statuses

Team:
- sees only assigned clients
- can work on media/posts/reports for assigned clients
- cannot see owner financials
- cannot change pricing
- cannot final-approve monthly reports unless explicitly allowed later

Operator:
- sees all operational data
- can approve monthly reports
- can manage risk/client health/alerts
- cannot change pricing or owner permissions

Owner:
- full access
- only role allowed to change pricing, packages, user roles, and high-level business settings

System:
- can insert/update derived data, logs, notifications, report summaries, publish statuses later
- should always create activity_logs for important automated changes

Check for:

- overly broad authenticated user policies
- missing client-safe view requirements
- owner-only fields not protected
- team access not scoped to team_client_assignments
- operator accidentally gaining pricing write access
- client direct SELECT on sensitive base tables
