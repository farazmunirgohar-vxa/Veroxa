PART 2 — CONTENT_CONCEPTS TABLE PLAN

Document table:

content_concepts

Fields:

- id uuid primary key default gen_random_uuid()
- client_id uuid not null references clients(id) on delete cascade
- media_asset_id uuid references media_assets(id) on delete set null
- content_angle text not null
- content_goal text check in ('awareness','engagement','conversion','branding','recovery')
- hook_style text check in ('question','bold_statement','story','stat','behind_scenes')
- cta_direction text check in ('visit','order','book','follow','share','none')
- status text not null default 'generated' check in ('generated','under_review','rejected','approved')
- generated_at timestamptz
- generated_by_agent text
- reviewed_by_user_id uuid references user_profiles(id) on delete set null
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

Notes:

- AI can suggest concepts later.
- Team/operator reviews before use.
- Client does not see raw concept workflow.

------------------------------------------------

PART 3 — DRAFT_SETS TABLE PLAN

Document table:

draft_sets

Fields:

- id uuid primary key default gen_random_uuid()
- concept_id uuid not null references content_concepts(id) on delete cascade
- generation_version integer not null default 1
- status text not null default 'generated' check in ('generated','under_review','needs_regeneration','approved','archived')
- team_note text
- generated_at timestamptz
- generated_by_agent text
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

------------------------------------------------

PART 4 — DRAFT_VARIANTS TABLE PLAN

Document table:

draft_variants

Fields:

- id uuid primary key default gen_random_uuid()
- draft_set_id uuid not null references draft_sets(id) on delete cascade
- variant_type text not null check in ('safe','engagement','sales')
- caption_body text not null
- hook_text text
- cta_text text
- hashtag_block text
- brand_voice_score integer
- status text not null default 'generated' check in ('generated','under_review','approved','archived','used')
- used_in_post_id uuid references posts(id) on delete set null
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

Notes:

- Client should not see raw draft variants in V1.
- Team uses drafts internally.
- Approved draft may connect to posts.

PART 7 — RLS PLAN

content_concepts:
- no client access in V1
- team can manage concepts for assigned clients
- operator can oversee all
- owner full access
- system/AI can insert generated concepts later

draft_sets:
- no client access in V1
- team can manage assigned client draft sets
- operator can review/override
- owner full access
- system/AI can insert generated drafts later

draft_variants:
- no client access in V1
- team can approve/use for assigned clients
- operator can review all
- owner full access
- system/AI can generate variants later

ai_agents:
- team/operator/owner can view relevant agent library
- owner controls enable/disable
- system writes last_run_at
- client no access
