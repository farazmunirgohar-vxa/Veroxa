PART 1 — REPLACE ARRAY-BASED TEAM ASSIGNMENTS WITH JOIN TABLE PLAN

Current issue:
The plan uses:

team_members.assigned_client_ids uuid[]

This is okay for demo thinking, but not ideal for real RLS.

Update the schema/RLS plan to use a dedicated join table:

team_client_assignments

Fields:
- id uuid primary key default gen_random_uuid()
- team_member_id uuid not null references team_members(id) on delete cascade
- client_id uuid not null references clients(id) on delete cascade
- assignment_role text not null default 'executor'
- is_active boolean not null default true
- created_at timestamptz not null default now()
- updated_at timestamptz not null default now()

Add a unique constraint:

unique(team_member_id, client_id)

Recommended assignment_role values:
- executor
- reviewer
- scheduler
- reporter
- lead

Update all references to team assignment logic.

Do not use assigned_client_ids uuid[] as the main real design.
