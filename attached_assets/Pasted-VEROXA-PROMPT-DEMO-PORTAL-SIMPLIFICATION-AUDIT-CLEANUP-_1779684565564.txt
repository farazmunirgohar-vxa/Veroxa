VEROXA PROMPT — DEMO PORTAL SIMPLIFICATION AUDIT + CLEANUP

Objective:
Simplify the current Veroxa demo portals so each role feels clear, premium, and focused.

Do NOT add new features.
Do NOT build backend.
Do NOT connect Supabase.
Do NOT connect OpenAI.
Do NOT add automation.
Do NOT add uploads.
Do NOT add publishing APIs.

This is a cleanup and clarity pass only.

Main goal:
Reduce unnecessary portal navigation, remove duplicate concepts, and make each role dashboard easier to understand.

------------------------------------------------

1. CLIENT PORTAL CLEANUP

Client Portal should only show these main navigation items:

- Dashboard
- Upload Media
- Calendar
- Reports
- Requests
- Account

Hide or remove from client nav:
- Workspace
- Activity
- Notifications as separate page
- Content Pipeline as a separate nav item
- Google as a separate nav item unless it is merged into Dashboard or Reports
- Any AI/internal/system language

Client should feel simple:
“Upload content, see reports, see upcoming posts, send requests.”

------------------------------------------------

2. TEAM PORTAL CLEANUP

Team Portal should only show:

- Dashboard
- Work Queue
- Media Review
- Drafts
- Scheduling
- Reports
- Alerts

Merge or hide overlapping items:
- Task Engine
- My Tasks
- Content Review
- Report Queue
- Performance
- Activity Feed

Keep Team focused on execution only.

------------------------------------------------

3. OPERATOR PORTAL CLEANUP

Operator Portal should only show:

- Command Center
- Client Health
- Alerts
- Reports Approval
- Media Library
- Team Oversight
- System Status

Hide or remove from main nav:
- Architecture
- Demo Controls
- Workflow Engine
- Operations Center
- Daily Digest
- Priority Board
- Action Center
- Content Operations
- Reporting Command
- AI Agents
- KPIs
- Activity
- Failed Posts as separate page unless merged into Alerts or Command Center

Operator should feel like oversight, risk, quality, and approvals.

------------------------------------------------

4. OWNER PORTAL CLEANUP

Owner Portal should only show:

- Executive Dashboard
- Revenue
- Client Health
- Critical Alerts
- AI/System Health
- Growth
- Settings

Hide or remove from main nav:
- BI Center
- Client Analytics
- Reporting Analytics
- Media Analytics
- Ops Intelligence
- Agent Workflow
- AI Agents Summary
- Permissions
- Automation Roadmap
- System Map
- Weekly Reports
- Monthly Reports
- Media Inventory
- Activity
- KPIs as separate page

Owner should feel high-level:
money, growth, retention, risk, system health.

------------------------------------------------

5. INTERNAL PAGES

Move development-only pages out of role navigation.

Development-only pages include:

- Architecture
- Demo Controls
- System Status
- Supabase Test
- Internal Integrations
- Real route placeholders

They may remain accessible through /demo/internal, but should not appear in Client, Team, Operator, or Owner main navigation.

------------------------------------------------

6. ROUTE SAFETY

Do not delete files unless safe.
Prefer hiding from navigation first.
Keep existing routes working.
Do not break App.tsx.
Do not break current demo pages.

------------------------------------------------

7. UI CLEANUP

Where pages have too many cards, reduce visual clutter.

Each main dashboard should have:

- Clear header
- 3–5 primary KPI cards max
- One main action section
- One alert/status section
- One recent activity or summary section

Avoid showing every possible metric at once.

------------------------------------------------

8. FINAL REPORT

At completion, report:

1. Navigation items before and after for each portal
2. Files modified
3. Routes affected
4. Pages hidden from navigation
5. Any duplicate pages that should be deleted later
6. Build result

Do not add new features.
This is a simplification and clarity pass only.
